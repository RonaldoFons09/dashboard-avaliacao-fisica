# Critério 2 — Nomenclatura Descritiva

## Definição

A escolha correta de nomes para **variáveis, funções, classes, módulos e bibliotecas** é fundamental para a compreensão do código.

Bons nomes tornam o código **autoexplicativo**, reduzindo a necessidade de comentários e evitando interpretações ambíguas.

---

## Princípio central

Código é lido **muito mais vezes** do que é escrito.

Quando os nomes são ruins, o leitor precisa **decifrar a intenção do autor**, gastando tempo e aumentando o risco de erro.

Por isso:
- Nomes devem **comunicar significado**, não economizar caracteres
- É preferível um nome **longo e claro** do que curto e enigmático

---

## O que este critério exige

Todo nome no código deve responder claramente:

- **O que isso representa?**
- **Qual é o papel disso no sistema?**

Evite:
- abreviações obscuras
- metáforas confusas
- letras soltas sem significado

---

## Boas práticas de nomenclatura

### ✔️ Variáveis

- Representam **dados**
- Devem ser substantivos claros

❌ Ruim:
```python
x = 10
```

✅ Bom:
```python
quantidade_itens = 10
```

---

### ✔️ Funções

- Representam **ações**
- Devem ser verbos ou frases verbais

❌ Ruim:
```python
def data():
    ...
```

✅ Bom:
```python
def calcular_data_entrega():
    ...
```

---

### ✔️ Classes

- Representam **entidades ou conceitos**
- Devem ser substantivos no singular

❌ Ruim:
```python
class Ger:
    ...
```

✅ Bom:
```python
class GerenciadorPedidos:
    ...
```

---

## Checklist de avaliação

Use as perguntas abaixo para validar a nomenclatura:

- [ ] O nome deixa claro o que o elemento representa?
- [ ] Uma pessoa nova entenderia o papel disso sem ler a implementação?
- [ ] O nome evita abreviações confusas ou genéricas?
- [ ] O nome expressa intenção, não apenas tipo ou forma?

Se alguma resposta for **não**, a nomenclatura deve ser revisada.

---

## Exemplos comparativos

### ❌ Nomes ruins

```python
x = 100
abc = True

def f(a, b):
    return a * b
```

Problemas:
- Não comunicam significado
- Exigem leitura do código para entendimento
- Aumentam a carga cognitiva

---

### ✅ Nomes descritivos

```python
valor_total = 100
possui_desconto = True

def calcular_valor_final(preco, quantidade):
    return preco * quantidade
```

Benefícios:
- Intenção clara
- Leitura fluida
- Código mais confiável

---

## Frase-guia

> **Se você precisa explicar o nome de algo, ele foi mal escolhido.**

---

## Objetivo do critério

Garantir que o código:

- comunique intenção
- reduza ambiguidade
- seja fácil de ler e manter
- funcione como documentação viva

Nomes bem escolhidos são uma das formas mais baratas e eficazes de melhorar a qualidade do código.
