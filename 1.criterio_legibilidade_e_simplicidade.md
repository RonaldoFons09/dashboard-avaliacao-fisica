# Critério 1 — Legibilidade e Simplicidade

## Definição

O código deve ser **fácil de ler, entender e manter**, mesmo por alguém que não o escreveu.

A prioridade é que a **intenção do código seja clara**, sem exigir esforço excessivo de interpretação ou conhecimento prévio do contexto.

---

## Princípio central

Programadores **não pensam como computadores**.
Eles interpretam código buscando:

- contexto
- intenção
- significado

Por isso, o código deve oferecer **abstrações claras**, usando nomes, estruturas e divisões que facilitem a compreensão humana.

---

## O que este critério exige

Cada trecho de código deve deixar evidente:

- **O que está sendo feito**
- **Por que está sendo feito**
- **Qual resultado é esperado**

A legibilidade deve ser priorizada acima de:

- otimizações prematuras
- truques de linguagem
- código excessivamente compacto

---

## Boas práticas associadas

- Funções pequenas e com uma única responsabilidade
- Nomes descritivos para funções, variáveis e parâmetros
- Estruturas simples (if/else claros, evitar lógica excessivamente aninhada)
- Código que se explica sozinho, sem depender de comentários para o básico

---

## Checklist de avaliação

Use as perguntas abaixo para avaliar se o critério está sendo atendido:

- [ ] Consigo entender o que a função faz apenas pelo nome?
- [ ] A intenção do código é clara sem ler cada linha em detalhe?
- [ ] Alguém novo no projeto entenderia esse código rapidamente?
- [ ] O código parece mais uma explicação do problema do que um truque técnico?

Se alguma resposta for **não**, o critério de legibilidade e simplicidade não está plenamente atendido.

---

## Exemplo prático

### ❌ Baixa legibilidade

```python
def calc(x, y):
    return x * 0.9 if y else x
```

Problemas:
- Nomes genéricos
- Regra de negócio implícita
- Intenção pouco clara

---

### ✅ Alta legibilidade

```python
def aplicar_desconto(valor, possui_desconto):
    if possui_desconto:
        return valor * 0.9
    return valor
```

Benefícios:
- Intenção explícita
- Regra de negócio clara
- Leitura imediata

---

## Frase-guia

> **Código legível é aquele que explica sua intenção sem precisar de comentários.**

---

## Objetivo do critério

Reduzir:
- erros
- tempo de leitura
- dependência de quem escreveu o código

E aumentar:
- clareza
- manutenibilidade
- qualidade geral do software
